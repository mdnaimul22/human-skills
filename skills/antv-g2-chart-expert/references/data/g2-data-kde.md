## Core Concepts

**KDE is a Data Transform, not a Mark Transform**

- Data transform configuration is in `data.transform`
- Kernel Density Estimation is a non-parametric statistical method to estimate the probability density function of a random variable
- Underlying implementation uses the [pdfast](https://www.npmjs.com/package/pdfast) library

**Output**: After processing, two fields are added to the data (default `y` and `size`), both of type array.

**⚠️ Key: `field` is the input, `as` is the output, encode must use the output field names**

```
kde: field='value' → 输出 as=['y','size'] → encode: { y: 'y', size: 'size' }
                      ↑ Regardless of the field name, encode always uses the values from `as`
```
## Minimum Viable Example

```javascript
import { Chart } from '@antv/g2';

const chart = new Chart({ container: 'container', width: 700, height: 400 });

chart.options({
  type: 'density',
  data: {
    type: 'fetch',
    value: 'https://assets.antv.antgroup.com/g2/species.json',
    transform: [
      {
        type: 'kde',
        field: 'y',              // Field for KDE
        groupBy: ['x', 'species'], // Grouping fields
        size: 20,                // Number of data points to generate
      },
    ],
  },
  encode: {
    x: 'x',
    y: 'y',
    color: 'species',
    size: 'size',
  },
  tooltip: false,
});

chart.render();
```

## Configuration Options

| Property | Description                                      | Type               | Default Value          | Required |
| -------- | ------------------------------------------------ | ------------------ | ---------------------- | -------- |
| field    | Data field for kernel density estimation (KDE)   | `string`           | -                      | ✓        |
| groupBy  | Field(s) to group data by, multiple can be specified | `string[]`         | -                      | ✓        |
| as       | Fields to store the results after KDE processing | `[string, string]` | `['y', 'size']`        | No       |
| min      | Minimum value for the processing range           | `number`           | Minimum value in data  | No       |
| max      | Maximum value for the processing range           | `number`           | Maximum value in data  | No       |
| size     | Number of data points generated by the algorithm; higher values result in finer density curves | `number`           | `10`                   | No       |
| width    | Determines the influence of each point on neighboring points; higher values result in smoother curves | `number`           | `2`                    | No       |

## Violin Plot

```javascript
chart.options({
  type: 'view',
  data: { type: 'fetch', value: 'https://assets.antv.antgroup.com/g2/species.json' },
  children: [
    {
      type: 'density',
      data: {
        transform: [
          { type: 'kde', field: 'y', groupBy: ['x', 'species'] },
        ],
      },
      encode: { x: 'x', y: 'y', series: 'species', color: 'species', size: 'size' },
      tooltip: false,
    },
    {
      type: 'boxplot',
      encode: { x: 'x', y: 'y', series: 'species', color: 'species', shape: 'violin' },
      style: { opacity: 0.5, strokeOpacity: 0.5, point: false },
    },
  ],
});
```

## Custom Parameters

```javascript
chart.options({
  type: 'density',
  data: {
    transform: [
      {
        type: 'kde',
        field: 'y',
        groupBy: ['x'],
        size: 30,             // Finer density curve
        width: 3,             // Smoother
        min: 0,               // Minimum value
        max: 8,               // Maximum value
        as: ['density_x', 'density_y'],  // Custom output field names
      },
    ],
  },
  encode: { x: 'x', y: 'density_x', size: 'density_y' },
});
```

## Common Errors and Fixes

### Error 1: Placing kde in mark transform

```javascript
// ❌ Incorrect: kde is a data transformation and cannot be placed in the transform of a mark
chart.options({
  type: 'density',
  data,
  transform: [{ type: 'kde', field: 'y' }],  // ❌ Incorrect location
});

// ✅ Correct: Place kde in data.transform
chart.options({
  type: 'density',
  data: {
    type: 'fetch',
    value: dataUrl,
    transform: [{ type: 'kde', field: 'y', groupBy: ['x'] }],  // ✅ Correct
  },
});
```

### Error 2: Missing groupBy

```javascript
// ❌ Error: groupBy is a required parameter
data: {
  transform: [{ type: 'kde', field: 'y' }],  // Missing groupBy
}

// ✅ Correct: Specify groupBy
data: {
  transform: [{ type: 'kde', field: 'y', groupBy: ['category'] }],
}
```

### Error 3: Zero Variance or Single Point Data Causes KDE Degeneration (Silent Blank)

The underlying KDE implementation (pdfast library) requires `min < max`. When all values in a group are identical (variance = 0) or there is only 1 data point, KDE produces NaN, and the group is not rendered.

```javascript
// ❌ Problem: Zero variance data
const data = [
  { group: 'A', value: 10 },           // Only 1 point
  { group: 'B', value: 20 },
  { group: 'B', value: 20 },           // All identical, min=max=20
  { group: 'B', value: 20 },
];
// KDE groupBy: ['group'] processes group B with min=max=20, produces NaN → not rendered

// ✅ Solution 1: Manually set min/max to ensure non-zero range
transform: [{
  type: 'kde',
  field: 'value',
  groupBy: ['group'],
  min: 0,    // Manually specified, ensures min ≠ max
  max: 100,
}]

// ✅ Solution 2: Switch chart type when data is insufficient
// KDE recommends at least 5-10 distinct values per group for meaningful density curves
// Single point or all identical values → Use scatter plot / box plot instead
```

### Error 4: Mismatch Between `encode` Fields and `as`

```javascript
// ❌ Error: Using default `as` but `encode` uses different field names
data: {
  transform: [{ type: 'kde', field: 'y', groupBy: ['x'] }],  // Default as: ['y', 'size']
}
encode: { y: 'density', size: 'density_size' },  // ❌ Field names do not match

// ✅ Correct: Using correct field names
encode: { y: 'y', size: 'size' },  // ✅ Using default output fields

// Or customize `as`
{
  transform: [{ type: 'kde', field: 'y', groupBy: ['x'], as: ['density', 'density_size'] }],
},
encode: { y: 'density', size: 'density_size' },  // ✅ Matches custom fields
```

### Error 5: Insufficient Data Volume Causes KDE Results to Be Empty or Invalid

KDE requires a sufficient number of data points to generate a valid density estimate. If there are too few data points within a group (e.g., less than or equal to 1), KDE cannot compute a valid result, which may cause the group to not be displayed.

```javascript
// ❌ Issue: Insufficient data in some groups
const data = [
  { species: 'setosa', x: 'SepalLength', y: 5.1 },
  { species: 'versicolor', x: 'SepalLength', y: 6.0 },
  { species: 'virginica', x: 'SepalLength' }, // Missing y value
];

// ✅ Solution: Ensure each group has enough valid data
const validData = [
  { species: 'setosa', x: 'SepalLength', y: 5.1 },
  { species: 'setosa', x: 'SepalLength', y: 5.0 },
  { species: 'versicolor', x: 'SepalLength', y: 6.0 },
  { species: 'versicolor', x: 'SepalLength', y: 6.2 },
  { species: 'virginica', x: 'SepalLength', y: 6.5 },
  { species: 'virginica', x: 'SepalLength', y: 6.3 },
];
```

### Error 6: Rendering Failure Due to Incorrect Chart Type or Encoding Settings

When using KDE in a composite view, ensure that each sub-chart has the correct type and encoding settings, especially the `series` mapping.

```javascript
// ❌ Error: Missing required encode field or improper type setting
children: [
  {
    type: 'density',
    data: { transform: [{ type: 'kde', field: 'y', groupBy: ['x', 'species'] }] },
    encode: { x: 'x', y: 'y', color: 'species', size: 'size' },
  },
  {
    type: 'boxplot',
    encode: { x: 'x', y: 'y', color: 'species', shape: 'violin' }, // Missing series mapping
  }
]

// ✅ Correct: Ensure all necessary fields are properly mapped
children: [
  {
    type: 'density',
    data: { transform: [{ type: 'kde', field: 'y', groupBy: ['x', 'species'] }] },
    encode: { x: 'x', y: 'y', series: 'species', color: 'species', size: 'size' },
  },
  {
    type: 'boxplot',
    encode: { x: 'x', y: 'y', series: 'species', color: 'species', shape: 'violin' },
  }
]
```

### Error 7: Insufficient Data Points After KDE Grouping Resulting in Blank Rendering

KDE requires each group to have enough data points to generate a valid density estimate. If there are too few data points after grouping (e.g., fewer than 2 valid values per group), KDE cannot compute a valid result, which may cause the entire chart to render as blank.

```javascript
// ❌ Issue: Too few data points per group after grouping
const insufficientData = [
  { species: 'setosa', x: 'SepalLength', y: 5.1 },
  { species: 'versicolor', x: 'SepalLength', y: 6.0 },
  { species: 'virginica', x: 'SepalLength', y: 6.5 },
];

// ✅ Solution: Ensure each group has enough valid data points
const sufficientData = [
  { species: 'setosa', x: 'SepalLength', y: 5.1 },
  { species: 'setosa', x: 'SepalLength', y: 5.0 },
  { species: 'setosa', x: 'SepalLength', y: 5.2 },
  { species: 'versicolor', x: 'SepalLength', y: 6.0 },
  { species: 'versicolor', x: 'SepalLength', y: 6.2 },
  { species: 'versicolor', x: 'SepalLength', y: 5.9 },
  { species: 'virginica', x: 'SepalLength', y: 6.5 },
  { species: 'virginica', x: 'SepalLength', y: 6.3 },
  { species: 'virginica', x: 'SepalLength', y: 6.7 },
];
```

### Error 8: Incorrectly Set `encode` When Using KDE with Density Chart Type

When using KDE as a data transformation, ensure that the `encode` correctly references the fields generated by KDE (defaulting to `y` and `size`). Otherwise, the chart may not render correctly.

```javascript
// ❌ Incorrect: Incorrectly referencing KDE-generated fields
chart.options({
  type: 'density',
  data: {
    transform: [{ type: 'kde', field: 'y', groupBy: ['x'] }]
  },
  encode: { x: 'x', y: 'originalY', size: 'originalSize' } // Incorrect field names
});

// ✅ Correct: Referencing KDE-generated fields
chart.options({
  type: 'density',
  data: {
    transform: [{ type: 'kde', field: 'y', groupBy: ['x'] }]
  },
  encode: { x: 'x', y: 'y', size: 'size' } // Correct field names
});
```

### Error 9: Improper KDE Grouping Field Selection Causes Rendering Anomalies

When using KDE, the selection of the `groupBy` field is crucial. Choosing the wrong field or failing to fully consider the data structure can result in density curves that do not meet expectations or are even completely invisible.

```javascript
// ❌ Incorrect: Improper groupBy field selection
chart.options({
  type: 'density',
  data: {
    transform: [
      {
        type: 'kde',
        field: 'y',
        groupBy: ['species'], // Only groups by species, ignoring different categories of the x field
      },
    ],
  },
  encode: {
    x: 'x',
    y: 'y',
    color: 'species',
    size: 'size',
  },
});

// ✅ Correct: Proper groupBy field selection
chart.options({
  type: 'density',
  data: {
    transform: [
      {
        type: 'kde',
        field: 'y',
        groupBy: ['x', 'species'], // Groups by both x and species
      },
    ],
  },
  encode: {
    x: 'x',
    y: 'y',
    color: 'species',
    size: 'size',
  },
});
```

### Error 10: KDE Output Fields Overwritten by Subsequent Operations

When fields generated by KDE (such as `y` and `size`) are modified or overwritten in subsequent data processing steps, it can lead to abnormal chart rendering.

```javascript
// ❌ Error: KDE output fields overwritten by subsequent transform
chart.options({
  type: 'density',
  data: {
    transform: [
      { type: 'kde', field: 'y', groupBy: ['x', 'species'] },
      { type: 'map', callback: (d) => ({ ...d, y: d.y.map(v => v * 2) }) } // Modifies the y field
    ],
  },
  encode: {
    x: 'x',
    y: 'y',
    color: 'species',
    size: 'size',
  },
});

// ✅ Correct: Use custom field names to avoid conflicts
chart.options({
  type: 'density',
  data: {
    transform: [
      { 
        type: 'kde', 
        field: 'y', 
        groupBy: ['x', 'species'],
        as: ['kdeY', 'kdeSize'] // Use custom field names
      },
      { type: 'map', callback: (d) => ({ ...d, y: d.y.map(v => v * 2) }) } // Does not affect KDE results
    ],
  },
  encode: {
    x: 'x',
    y: 'kdeY', // Use custom field generated by KDE
    color: 'species',
    size: 'kdeSize',
  },
});
```
